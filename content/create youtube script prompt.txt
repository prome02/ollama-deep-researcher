針對主題以繁體中文寫youtube節目的10分鐘劇本與旁白。
搜尋資訊過程以英文資料為主, 中文資料為輔。
最終輸出json物件。

文本內容至少四個段落，前面兩段可以埋下文章的"鉤子"，後兩段確定會回應所有"鉤子"；
第一段先對主題進行背景介紹。
第二段進行主題的主要描述，特別著重描述事件發生過程。包括與事件相關的人員名字與背景簡介、關聯技術、特殊裝置或物件、發生時間、地點等等。 第三段對第二段提出反思，或是邏輯漏洞、相異看法，進行觀點的發想或辯證。
第四段收斂結果，對二三段的結果並陳，評估哪些觀點是比較值得信任的；對於不信任的觀點做出解釋。或許加上延伸話題，並總結文章作為結束。

文章最後要加上資料來源。 可附加關聯的媒體連結，例如：圖片、影像、聲音等。

////優化後

以英文(en)、繁體中文(zh)撰寫 面對一般成年人聽眾的至少10 分鐘的 YouTube 劇本與旁白，根據前面的討論，加上額外英文為主、中文為輔的資料搜尋。以 JSON 輸出，包含以下段落結構：

主題背景（埋下鉤子）

事件詳述：關鍵人物、技術、裝置、時間地點等資訊（持續埋鉤子）

觀點反思或辯證：包含矛盾、邏輯漏洞、不同立場

結論：整合觀點、判斷可信度、提供延伸話題或總結

加入資料來源與可選媒體連結（如圖片、影片、音檔）。

json物件格式舉例說明("content"內的元素的"Narration"內容不宜太長，可以使"speak_instructions"與"prompt"的描述更精確細緻)：

{
  "title":{
    "en":"英文文章標題",
    "zh":"繁體中文文章標題"
  },
  "introduction":{
    "en":"英文youtube簡介",
    "zh":"繁體中文youtube簡介"
  },

  "music":"整體配樂的AI生成音樂提示詞",
  "content": [
    {
      "caption": {
        "en":"英文段落標題",
        "zh":"繁體中文段落標題"
      },
      "description": "段落描述或時間標記",
      "Narration": {
        "en":"英文的旁白",
        "zh":"en的繁體中文旁白"
      },
      "speak_instructions":"text-to-speech提示詞，帶入情感標記",
      "prompt":"AI 生成圖片的提示詞",
	    "music_prompt":"AI 生成音樂的提示詞，非必要"
    },
    ...
  ],
  "resource": ["資料來源1","資料來源2"],
  "note": "繁體中文的 其他注意事項"
}
